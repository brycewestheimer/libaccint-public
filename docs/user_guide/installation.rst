.. _installation:

Installation Guide
==================

This guide covers installation of LibAccInt on all supported platforms.

System Requirements
-------------------

Minimum Requirements
~~~~~~~~~~~~~~~~~~~~

- **Operating System**: Linux (Ubuntu 20.04+, RHEL 8+), macOS 12+, Windows 10+ (WSL2)
- **Compiler**: C++20 support required

  - GCC 11.0 or later
  - Clang 14.0 or later
  - MSVC 19.30 or later (Visual Studio 2022)

- **CMake**: 3.25 or later
- **Memory**: 4 GB RAM minimum, 8 GB recommended
- **Disk**: 500 MB for build, 100 MB installed

Optional Dependencies
~~~~~~~~~~~~~~~~~~~~~

- **CUDA Toolkit 11.0+**: For NVIDIA GPU acceleration
- **OpenMP**: For CPU parallelization (usually included with compiler)
- **Python 3.8+**: For Python bindings
- **Doxygen 1.9+**: For building documentation
- **Sphinx 4.0+**: For user documentation

Build from Source
-----------------

Linux
~~~~~

.. code-block:: bash

   # Install dependencies (Ubuntu/Debian)
   sudo apt update
   sudo apt install build-essential cmake git

   # Clone repository
   git clone https://github.com/yourusername/libaccint.git
   cd libaccint

   # Configure and build (CPU only)
   cmake --preset cpu-release
   cmake --build --preset cpu-release

   # Run tests
   ctest --test-dir build/cpu-release

   # Install system-wide (optional)
   sudo cmake --install build/cpu-release --prefix /usr/local

macOS
~~~~~

.. code-block:: bash

   # Install dependencies via Homebrew
   brew install cmake llvm

   # Clone repository
   git clone https://github.com/yourusername/libaccint.git
   cd libaccint

   # Configure with Homebrew Clang (for OpenMP support)
   cmake --preset cpu-release \
       -DCMAKE_C_COMPILER=$(brew --prefix llvm)/bin/clang \
       -DCMAKE_CXX_COMPILER=$(brew --prefix llvm)/bin/clang++
   cmake --build --preset cpu-release

Windows (WSL2)
~~~~~~~~~~~~~~

We recommend using WSL2 with Ubuntu for Windows development:

.. code-block:: bash

   # Install WSL2 (from PowerShell as Administrator)
   wsl --install -d Ubuntu

   # Then follow Linux instructions above

CUDA Build
~~~~~~~~~~

For NVIDIA GPU acceleration:

.. code-block:: bash

   # Ensure CUDA toolkit is installed and in PATH
   nvcc --version

   # Configure with CUDA
   cmake --preset cuda-release
   cmake --build --preset cuda-release

   # Verify GPU detection
   ctest --test-dir build/cuda-release -R cuda

Spack Build
~~~~~~~~~~~

LibAccInt also ships an in-tree Spack repository for HPC-oriented builds.

.. code-block:: bash

   # Add the local repository once
   spack repo add ./packaging/spack

   # CPU/OpenMP build
   spack install libaccint@main +openmp

   # MPI-enabled build
   spack install libaccint@main +mpi

   # CUDA-enabled build
   spack install libaccint@main +cuda cuda_arch=90

   # Build directly from the current checkout
   spack dev-build -d . libaccint@main +openmp

The Spack recipe disables CMake ``FetchContent`` and expects dependencies such
as ``nlohmann-json``, ``eigen``, ``googletest``, Python packages, CUDA,
and MPI to come from Spack variants and dependency resolution.

Release Source Archive
~~~~~~~~~~~~~~~~~~~~~~

To generate the deterministic source archive used by packaging and release
artifacts:

.. code-block:: bash

   ./packaging/release/build_source_release.sh

This produces ``dist/libaccint-<version>.tar.gz`` plus a matching
``.sha256`` file. The Conda and Spack recipes track that release asset rather
than GitHub's autogenerated tag archives.

CMake Options
-------------

+----------------------------+---------+----------------------------------------+
| Option                     | Default | Description                            |
+============================+=========+========================================+
| LIBACCINT_USE_CUDA         | AUTO    | Enable CUDA backend (AUTO/ON/OFF)      |
+----------------------------+---------+----------------------------------------+
| LIBACCINT_BUILD_TESTS      | ON      | Build test suite                       |
+----------------------------+---------+----------------------------------------+
| LIBACCINT_BUILD_EXAMPLES   | ON      | Build example programs                 |
+----------------------------+---------+----------------------------------------+
| LIBACCINT_USE_OPENMP       | ON      | Enable OpenMP parallelization          |
+----------------------------+---------+----------------------------------------+
| LIBACCINT_BUILD_PYTHON     | OFF     | Build Python bindings                  |
+----------------------------+---------+----------------------------------------+
| LIBACCINT_BUILD_DOCS       | OFF     | Build documentation                    |
+----------------------------+---------+----------------------------------------+

Available Presets
~~~~~~~~~~~~~~~~~

+----------------+------------------------------------------+
| Preset         | Description                              |
+================+==========================================+
| cpu-debug      | CPU build with debug symbols             |
+----------------+------------------------------------------+
| cpu-release    | CPU build with optimizations             |
+----------------+------------------------------------------+
| cuda-debug     | CUDA + CPU debug build                   |
+----------------+------------------------------------------+
| cuda-release   | CUDA + CPU release build                 |
+----------------+------------------------------------------+

Python Bindings
---------------

To build and install the Python bindings:

.. code-block:: bash

   # Configure with Python bindings
   cmake --preset cpu-release -DLIBACCINT_BUILD_PYTHON=ON
   cmake --build --preset cpu-release

   # Install in development mode
   cd python
   pip install -e .

   # Verify installation
   python -c "import libaccint; print(libaccint.__version__)"

Using in Your Project
---------------------

CMake Integration
~~~~~~~~~~~~~~~~~

LibAccInt provides CMake package configuration for easy integration:

.. code-block:: cmake

   # In your CMakeLists.txt
   find_package(LibAccInt REQUIRED)
   target_link_libraries(your_target LibAccInt::libaccint)

If LibAccInt is not installed system-wide, specify its location:

.. code-block:: bash

   cmake -DLibAccInt_DIR=/path/to/libaccint/build/cpu-release ..

pkg-config
~~~~~~~~~~

LibAccInt also provides pkg-config files:

.. code-block:: bash

   export PKG_CONFIG_PATH=/path/to/libaccint/build/cpu-release:$PKG_CONFIG_PATH
   pkg-config --cflags --libs libaccint

Troubleshooting
---------------

Common Issues
~~~~~~~~~~~~~

**CMake version too old**
   Upgrade CMake to 3.25+:

   .. code-block:: bash

      pip install cmake --upgrade

**C++20 not supported**
   Upgrade your compiler. On Ubuntu:

   .. code-block:: bash

      sudo apt install g++-11
      export CXX=g++-11

**CUDA not found**
   Ensure CUDA is in your PATH:

   .. code-block:: bash

      export PATH=/usr/local/cuda/bin:$PATH
      export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH

**OpenMP not found on macOS**
   Use Homebrew's LLVM which includes OpenMP:

   .. code-block:: bash

      brew install llvm
