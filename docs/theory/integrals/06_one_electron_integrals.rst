One-Electron Integrals
======================

Scope
-----

This chapter covers the main one-electron AO integrals:

- overlap :math:`S_{\mu\nu}`,
- kinetic :math:`T_{\mu\nu}`,
- nuclear attraction :math:`V_{\mu\nu}`,
- optional multipole-like operators.

All formulas are written at primitive level and then contracted.

Overlap Integrals
-----------------

Definition:

.. math::

   S_{\mu\nu}=\int\chi_\mu(\mathbf{r})\chi_\nu(\mathbf{r})\,d\mathbf{r}

For primitive s-functions:

.. math::

   S_{ss}=\left(\frac{\pi}{\zeta}\right)^{3/2}
   \exp\left(-\frac{\alpha\beta}{\zeta}R_{AB}^2\right)

Higher angular momentum terms are generated by recurrence (chapter
:doc:`07_recurrence_relations`).

Kinetic Energy Integrals
------------------------

Definition:

.. math::

   T_{\mu\nu}=\braket{\mu}{-\frac{1}{2}\nabla^2\nu}

For Cartesian Gaussians, applying :math:`\nabla^2` to polynomial-Gaussian form
reduces kinetic integrals to linear combinations of overlap-like integrals with
shifted angular momentum.

For primitive Cartesian tuples :math:`\mathbf{a},\mathbf{b}`, a commonly used
identity is:

.. math::
   :label: eq-onee-kinetic-overlap

   T_{\mathbf{a}\mathbf{b}} =
   \beta(2L_b+3)S_{\mathbf{a}\mathbf{b}}
   - 2\beta^2\sum_{j\in\{x,y,z\}}S_{\mathbf{a},\,\mathbf{b}+2\mathbf{1}_j}
   - \frac{1}{2}\sum_{j\in\{x,y,z\}}b_j(b_j-1)S_{\mathbf{a},\,\mathbf{b}-2\mathbf{1}_j}

where :math:`L_b=b_x+b_y+b_z` and :math:`S` is the overlap class.

This allows kinetic kernels to reuse overlap recurrence intermediates with known
coefficient prefactors.

Nuclear Attraction Integrals
----------------------------

Definition:

.. math::

   V_{\mu\nu}=\sum_C -Z_C\int
   \chi_\mu(\mathbf{r})\frac{1}{\lVert\mathbf{r}-\mathbf{C}\rVert}\chi_\nu(\mathbf{r})
   d\mathbf{r}

For primitive s-functions on centers `A,B`, attraction to nucleus `C` is

.. math::
   :label: eq-onee-vss

   V_{ss}^{(C)} = -Z_C\frac{2\pi}{\zeta}K_{AB}F_0(T_C),
   \qquad
   T_C=\zeta\lVert\mathbf{P}-\mathbf{C}\rVert^2

Higher angular momentum is generated with one-electron VRR
(:eq:`eq-rr-vrr-one-electron`). In Hermite form, a compact expression is

.. math::

   V_{\mathbf{a}\mathbf{b}}^{(C)} =
   -Z_C\frac{2\pi}{\zeta}K_{AB}
   \sum_{tuv}
   E_t^{a_xb_x}E_u^{a_yb_y}E_v^{a_zb_z}
   R_{tuv}(T_C)

where :math:`R_{tuv}` are Hermite Coulomb intermediates and :math:`E` are
Hermite expansion coefficients.

Primitive forms therefore depend on Boys values :math:`F_m(T)` with

.. math::

   T = \zeta\lVert\mathbf{P}-\mathbf{C}\rVert^2

Hence robust Boys evaluation is mandatory for stable nuclear attraction kernels
(:doc:`05_boys_functions`).

Multipole and Related Operators
-------------------------------

Many engines also compute:

- dipole: :math:`\braket{\mu}{r_\xi\nu}`,
- quadrupole: :math:`\braket{\mu}{r_\xi r_\eta\nu}`,
- momentum and angular momentum operators.

These usually reuse overlap/nuclear building blocks with shifted moments.

Algorithmic Structure
---------------------

Typical shell-pair algorithm:

1. Build primitive pair invariants.
2. For each operator class, evaluate base moments/Boys values.
3. Apply RR or Hermite recursions to obtain Cartesian block.
4. Contract primitives to AO block.
5. Apply spherical transform if needed.

Implementation-grade pseudocode:

.. code-block:: cpp

   void one_electron_shell_pair(const ShellPair& sp,
                                const std::vector<Atom>& atoms,
                                OneEBlock& S, OneEBlock& T, OneEBlock& V) {
     zero(S); zero(T); zero(V);

     for (PrimPair pq : sp.primitive_pairs()) {
       Invariants inv = build_pair_invariants(pq);      // zeta, P, K_AB, etc.

       OverlapBlock Sop = overlap_block_cartesian(pq, inv);
       KineticBlock Kop = kinetic_from_overlap(pq, Sop, inv); // eq-onee-kinetic-overlap

       AttractionBlock Vop;
       for (const Atom& A : atoms) {
         double Tc = inv.zeta * norm2(inv.P - A.pos);
         BoysVals F = boys_eval(required_mmax(sp), Tc);
         Vop += attraction_block_cartesian(pq, inv, A, F);
       }

       contract_accumulate(pq, Sop, S);
       contract_accumulate(pq, Kop, T);
       contract_accumulate(pq, Vop, V);
     }
   }

Screening for one-electron classes can exploit pair-distance prefactors and
atom-dependent bounds for nuclear terms.

Complexity
----------

Dense AO matrices scale as :math:`O(N^2)` in AO count, with additional factor
for number of nuclei in attraction integrals. In practical systems, shell-pair
screening and distance decay remove a large fraction of negligible pairs.

Validation Strategy
-------------------

1. Compare overlap and kinetic matrices to reference library output.
2. Verify Hermiticity and positive-semidefinite behavior of overlap.
3. Validate nuclear attraction with per-atom decomposition checks.
4. Cross-check gradients with finite differences if derivative kernels are used.
